## Why Free?

Hugging Face provides free access to open-source AI models as a community service. The model (Mistral-7B) is:
- Open source and freely available
- Powerful enough for nutrition estimation
- Running on Hugging Face's infrastructure at no cost
- Rate-limited during peak usage but generally reliable

## Limitations

Because this is a free service:
- **May be slower** during peak hours (when many people are using it globally)
- **May have rate limits** if too many requests come at once
- **Model loading time**: First request might take 10-20 seconds if the model needs to wake up
- **Less consistent format**: May not always return data in perfect JSON format (but still helpful!)

## Tips for Best Results

1. **Be specific about portions**: "6 oz chicken" is better than "some chicken"
2. **Mention cooking method**: "grilled", "fried", "baked" affects calories
3. **Include all ingredients**: Don't forget oils, sauces, and condiments
4. **Ask follow-up questions**: The AI remembers recent context
5. **Be patient**: First query might be slower as the model loads

## Privacy

- Your meal descriptions are sent to Hugging Face's API for processing
- Hugging Face doesn't store queries or train on them
- All estimates stay in your browser's local storage
- No account or personal information required
- Perfect for privacy-conscious users!

## Troubleshooting

**"Sorry, I encountered an error"**:
- Hugging Face API might be experiencing high traffic
- Wait 30 seconds and try again
- Model might be loading (can take 10-20 seconds on first use)

**Very slow response**:
- Model is loading from cold start
- High global traffic on free tier
- This is normal - be patient!

**Odd or incomplete responses**:
- Try rephrasing your question
- Add more details about the meal
- The model might need clearer instructions

**Model loading message**:
- If you see "Model is loading", wait 20-30 seconds
- This only happens after periods of inactivity
- Subsequent requests will be much faster
